🎯 Recommended Auth Pattern for Salah Buddy

Use Supabase Auth with Custom Family Account Logic

Database Schema:

-- Parents/Guardians table
CREATE TABLE parents (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Kids table (linked to parent)
CREATE TABLE kids (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  parent_id UUID REFERENCES parents(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  age INTEGER,
  avatar TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Auth users link to parents
CREATE TABLE auth_parent_link (
  auth_user_id UUID REFERENCES auth.users(id) PRIMARY KEY,
  parent_id UUID REFERENCES parents(id) NOT NULL
);

📱 Authentication Flow

  1. Parent Sign-up: Parent creates account with email (magic link or password)
  2. Add Kids: Parent adds kid profiles (no passwords required)
  3. Kid Login: Parent logs in → Selects which kid to track for → Kid sees their dashboard
  4. Security: Use Supabase Row Level Security (RLS) to ensure parents only see their own kids' data
💡 Alternative Pattern: Magic links for kids - Each kid gets a unique magic link sent to parent's email. Parent clicks to "log in as" that kid. Safer than passwords for young children.

📊 Database Schema for Gamification

-- Activities table (tracks prayers and fasting)
CREATE TABLE activities (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  kid_id UUID REFERENCES kids(id) ON DELETE CASCADE,
  activity_type TEXT NOT NULL, -- 'salah' or 'fasting'
  activity_name TEXT, -- 'fajr', 'dhuhr', etc.
  completed_at TIMESTAMP DEFAULT NOW(),
  points INTEGER DEFAULT 10
);

-- Streaks table
CREATE TABLE streaks (
  kid_id UUID PRIMARY KEY REFERENCES kids(id) ON DELETE CASCADE,
  activity_type TEXT NOT NULL,
  current_streak INTEGER DEFAULT 0,
  longest_streak INTEGER DEFAULT 0,
  last_activity_date DATE,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Badges table
CREATE TABLE kid_badges (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  kid_id UUID REFERENCES kids(id) ON DELETE CASCADE,
  badge_type TEXT NOT NULL, -- '7-day-streak', 'ramadan-champion', etc.
  earned_at TIMESTAMP DEFAULT NOW()
);

-- Leaderboard view (calculated in real-time)
CREATE VIEW leaderboard AS
SELECT
  k.id,
  k.name,
  k.avatar,
  COUNT(a.id) as total_activities,
  SUM(a.points) as total_points,
  MAX(s.current_streak) as best_streak
FROM kids k
LEFT JOIN activities a ON k.id = a.kid_id
LEFT JOIN streaks s ON k.id = s.kid_id
GROUP BY k.id
ORDER BY total_points DESC;

⚡ Real-time Leaderboard Implementation

// components/Leaderboard.tsx
'use client'
import { useEffect, useState } from 'react'
import { createClient } from '@/lib/supabase/client'

export default function Leaderboard() {
  const [leaders, setLeaders] = useState([])
  const supabase = createClient()

  useEffect(() => {
    // Initial load
    loadLeaderboard()

    // Subscribe to real-time updates
    const channel = supabase
      .channel('leaderboard-updates')
      .on('postgres_changes',
        { event: '*', schema: 'public', table: 'activities' },
        () => loadLeaderboard()
      )
      .subscribe()

    return () => {
      supabase.removeChannel(channel)
    }
  }, [])

  const loadLeaderboard = async () => {
    const { data } = await supabase
      .from('leaderboard')
      .select('*')
      .limit(10)
    setLeaders(data || [])
  }

  return (
    

      {leaders.map((leader, index) => (
        

          #{index + 1}
          {leader.name}
          {leader.total_points} points
        

      ))}
    

  )
}

7️⃣ COPPA Compliance & Legal Considerations

⚠️ CRITICAL: If you're collecting data from kids under 13 and have users in the US, you MUST comply with COPPA (Children's Online Privacy Protection Act).

📜 What is COPPA?

COPPA is a US federal law that requires companies to obtain verifiable parental consent before collecting, using, or disclosing personal information from children 13 and under.

🔄 Major 2025 Updates

  • Final amendments published April 22, 2025
  • Effective date: June 23, 2025
  • Full compliance required by April 22, 2026
  • Penalties: Up to $50,120 per violation

✅ COPPA Requirements for Salah Buddy

1. Verifiable Parental Consent

Before collecting ANY personal information from kids, you must obtain consent from parents via methods like:

  • ✅ Email + Confirmation: Email parent, they click link to confirm
  • ✅ Credit Card Verification: Small charge (refunded) to verify adult
  • ❌ Checkbox: NOT sufficient
  • ❌ Age Gate: NOT sufficient without parent verification

2. Direct Notice to Parents

You must provide clear notice to parents about:

  • What personal information you collect from kids
  • How you use that information
  • Whether you share it with third parties
  • Parents' right to review and delete their child's data

3. Data Minimization

  • Only collect what's necessary (name, age for tracking)
  • Don't require more info than needed to participate
  • Don't sell or share kids' data with third parties

4. Parental Rights

Parents must be able to:

  • Review their child's personal information
  • Delete their child's information
  • Refuse further collection or use

🎯 Recommended COPPA Strategy for Salah Buddy

Parent-Controlled Accounts (COPPA-Friendly)

Flow:

  1. Parent Creates Account: Parent signs up with their email (Supabase Auth)
  2. Parent Adds Kids: Parent creates kid profiles (just name, age, optional avatar)
  3. No Kid PII Collected: Kids don't have email addresses, passwords, or direct accounts
  4. Parent Controls Everything: Parent can view, edit, delete all kid data
  5. Privacy Policy: Clear statement that parent controls all data

Why this works:

  • ✅ Parent's email is the only PII collected (adult, not subject to COPPA)
  • ✅ Kids are just "profiles" under parent account
  • ✅ Parent has full control and visibility
  • ✅ No direct data collection from kids
  • ✅ Parent consent is implicit in creating kid profiles

🚨 What NOT to Do

  • ❌ Let kids create their own accounts with email addresses
  • ❌ Collect kids' email, phone numbers, or location
  • ❌ Allow public profiles or chat between kids
  • ❌ Use behavioral advertising or tracking cookies on kids
  • ❌ Share kids' data with analytics providers without disclosure

📄 Required Legal Pages

  • Privacy Policy: Must explain data practices in clear language
  • Terms of Service: Age requirements, parent responsibilities
  • COPPA Notice: Dedicated page explaining COPPA compliance
